\abstract

\keywords{BACKEND-ПЛАТФОРМА, МИКРОСЕРВИСНАЯ АРХИТЕКТУРА, ТРЕНЕР, СПОРТСМЕН, ТРЕНИРОВОЧНЫЙ ПРОЦЕСС, REST API, NATS JETSTREAM, POSTGRESQL, АНАЛИТИКА, AI-РЕКОМЕНДАЦИИ}

Выпускная квалификационная работа посвящена разработке backend-части открытой микросервисной платформы CoachLink для автоматизации взаимодействия в системе <<тренер-спортсмен>>. В работе рассматривается проблема разрозненного обмена тренировочными заданиями и отчетами через мессенджеры, электронные таблицы и произвольные форматы данных.

Целью работы является создание серверной платформы, обеспечивающей регистрацию пользователей, разграничение ролей тренера и спортсмена, управление связями и тренировочными группами, создание тренировочных планов, назначение тренировок, прием стандартизированных отчетов, формирование уведомлений, агрегацию тренировочных данных и выдачу AI-рекомендаций по последним отчетам спортсмена.

В рамках работы спроектирована микросервисная архитектура, включающая API Gateway, Auth Service, User Service, Training Service, Notification Service, Analytics Service и AI Service. Сервисы реализованы на языке Go, используют PostgreSQL для хранения данных, NATS JetStream для событийного взаимодействия и OpenAPI для документирования публичных интерфейсов. AI-рекомендации формируются через Ollama и рассматриваются как вспомогательный инструмент тренера.

Предметная реализация первой версии ориентирована на легкую атлетику, однако архитектура платформы не ограничивается этим видом спорта и допускает дальнейшее расширение структуры тренировок, отчетов, шаблонов и метрик для других спортивных дисциплин.
