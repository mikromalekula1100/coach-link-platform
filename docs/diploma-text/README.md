# Текст Диплома По Backend-Части CoachLink

В этой папке разложен подробный черновик диплома по отдельным частям. Файлы можно использовать как основу для финального документа и постепенно приводить к требованиям кафедры по оформлению.

## Порядок Файлов

1. `01-introduction.md` - введение: актуальность, объект, предмет, цель, задачи, ограничения и структура работы.
2. `02-chapter-1-analytical-review.md` - аналитический обзор существующих подходов и формирование требований к backend.
3. `03-chapter-2-design.md` - проектная часть: архитектура, сервисы, данные, сценарии, API и обоснование технологий.
4. `04-chapter-3-implementation-testing.md` - реализация и тестирование backend-сервисов.
5. `05-conclusion.md` - заключение, итоги, ограничения и направления развития.
6. `06-appendices-and-artifacts.md` - приложения: таблицы, диаграммы, API-примеры и тестовые артефакты.

## Важные Ограничения Для Финального Текста

Backend является зоной ответственности автора диплома. Мобильное приложение нужно упоминать только как внешний REST-клиент.

Предметная реализация первой версии ориентирована на легкую атлетику, потому что автор занимается этим видом спорта и понимает его тренировочные данные. Это не архитектурное ограничение платформы: поддержку других видов спорта следует описывать как направление расширения модели тренировок, отчетов, шаблонов и метрик.

AI Service в текущей реализации предоставляет одну публичную ручку:

```text
POST /api/v1/ai/athletes/{id}/recommendations
```

Она формирует рекомендации по последним отчетам спортсмена через Ollama. Расширенные AI-сценарии нужно описывать только как будущее развитие.

## Что Нужно Сделать Перед Финальной Сдачей

Перед переносом текста в финальный документ нужно:

- сверить названия endpoints с `docs/api/openapi.yaml`;
- сверить архитектурные формулировки с `docs/architecture.md`;
- выполнить актуальные тесты и зафиксировать только реальные результаты;
- привести диаграммы к единому стилю оформления;
- заменить осторожные формулировки о тестировании на конкретные результаты запусков;
- отредактировать текст под методические требования кафедры.
